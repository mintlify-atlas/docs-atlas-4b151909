Apache Amoro is a Lakehouse management system built on open data lake formats. It brings pluggable and self-managed features to your Lakehouse, providing an out-of-the-box data warehouse experience while helping data platforms build infra-decoupled, stream-and-batch-fused, lake-native architecture.

## What is Amoro?

Amoro's central component is **AMS** (Amoro Management Service), which manages table metadata and provides self-optimizing and catalog services across multiple open table formats.

<CardGroup cols={3}>
  <Card title="Self-Optimizing" icon="arrows-rotate">
    Continuously compacts small files, removes expired data, and optimizes table layout — automatically and asynchronously, with no impact on reads or writes.
  </Card>
  <Card title="Multi-Format" icon="table">
    Manage Iceberg, Mixed-Iceberg, Mixed-Hive, and Paimon tables through a single unified interface.
  </Card>
  <Card title="Catalog Service" icon="database">
    Unified catalog service for all compute engines, compatible with Hive Metastore and AWS Glue.
  </Card>
  <Card title="Multi-Engine" icon="gears">
    Full integration with Apache Flink (streaming + batch), Apache Spark (batch + overwrite), and Trino (read + overwrite).
  </Card>
  <Card title="Web Dashboard" icon="chart-bar">
    Manage catalogs, tables, optimizers, and view real-time metrics through the built-in web UI.
  </Card>
  <Card title="REST API" icon="code">
    OpenAPI-compliant HTTP API for programmatic access to all AMS management features.
  </Card>
</CardGroup>

## Supported table formats

| Format | Streaming Write | CDC Read | Self-Optimizing | Hive Compatible |
|---|---|---|---|---|
| Iceberg | via Flink | via position deletes | Yes | No |
| Mixed-Iceberg | Yes (LogStore) | Yes (ChangeStore) | Yes | No |
| Mixed-Hive | Yes (LogStore) | Yes (ChangeStore) | Yes | Yes |
| Paimon | via Paimon engines | — | Metadata view | — |

## Getting started

<Steps>
  <Step title="Deploy AMS">
    Download the Amoro distribution and start the Amoro Management Service. See the [Deployment guide](/admin-guides/deployment).
  </Step>
  <Step title="Create a catalog">
    Register your storage environment (Hive Metastore, Hadoop, or S3) as an Amoro catalog. See [Managing catalogs](/admin-guides/managing-catalogs).
  </Step>
  <Step title="Create and manage tables">
    Create Iceberg or Mixed-format tables and let Amoro handle self-optimizing automatically. See [Using tables](/user-guides/using-tables).
  </Step>
  <Step title="Connect your compute engine">
    Configure Flink, Spark, or Trino to use the Amoro catalog for reads and writes. See [Engine Integration](/engines/flink/flink-get-started).
  </Step>
</Steps>
