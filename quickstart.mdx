This guide walks you through downloading Amoro, starting the Amoro Management Service (AMS), and creating your first catalog and table using the Dashboard.

## Prerequisites

- **Java 11** is required. Verify your installation:

```bash
java -version
```

## Steps

<Steps>
  <Step title="Download and extract">
    Download the latest release package from the [Apache Amoro download page](https://amoro.apache.org/download/).

    ```bash
    tar -xzf apache-amoro-x.y.z-bin.tar.gz
    cd amoro-x.y.z
    ```

    Replace `x.y.z` with the actual release version.
  </Step>

  <Step title="Review the configuration">
    The default configuration at `conf/config.yaml` is ready to use out of the box for local development. It starts AMS with an embedded Derby database and binds all services to localhost.

    ```yaml
    ams:
      admin-username: admin
      admin-password: admin
      server-bind-host: "0.0.0.0"
      server-expose-host: "127.0.0.1"

      thrift-server:
        table-service:
          bind-port: 1260   # Compute engines connect here
        optimizing-service:
          bind-port: 1261   # Optimizers connect here

      http-server:
        bind-port: 1630     # Dashboard and REST API

      database:
        type: derby
        jdbc-driver-class: org.apache.derby.jdbc.EmbeddedDriver
        url: jdbc:derby:/tmp/amoro/derby;create=true
    ```

    <Note>
      The default configuration uses an embedded Apache Derby database stored at `/tmp/amoro/derby`. Derby is suitable for local development and evaluation, but is not recommended for production use.
    </Note>

    <Tip>
      For production deployments, switch to MySQL or PostgreSQL by updating the `database` section in `conf/config.yaml`. Create an empty database schema first — AMS creates all tables automatically on first start.

      ```yaml
      ams:
        database:
          type: mysql
          jdbc-driver-class: com.mysql.cj.jdbc.Driver
          url: jdbc:mysql://127.0.0.1:3306/amoro?useUnicode=true&characterEncoding=UTF8&autoReconnect=true&useAffectedRows=true&allowPublicKeyRetrieval=true&useSSL=false
          username: root
          password: root
          auto-create-tables: true
      ```

      See [Deployment](/admin-guides/deployment) for full production configuration guidance.
    </Tip>
  </Step>

  <Step title="Start AMS">
    Run the startup script from the Amoro installation directory:

    ```bash
    bin/ams.sh start
    ```

    To restart or stop AMS:

    ```bash
    bin/ams.sh restart
    bin/ams.sh stop
    ```
  </Step>

  <Step title="Log in to the Dashboard">
    Open [http://localhost:1630](http://localhost:1630) in a browser. You should see the AMS login page.

    Log in with the default credentials:

    | Field | Value |
    |-------|-------|
    | Username | `admin` |
    | Password | `admin` |

    <Warning>
      Change the default admin password before exposing AMS to any network. Update `ams.admin-password` in `conf/config.yaml` and restart AMS.
    </Warning>
  </Step>

  <Step title="Create a catalog">
    In the Dashboard, navigate to **Catalogs** in the left sidebar and click **Create Catalog**.

    You will be prompted to configure:

    - **Name** — A unique identifier for the catalog (e.g., `my_catalog`).
    - **Metastore** — The metadata backend. Choose from Hive Metastore, Hadoop, or AWS Glue.
    - **Table format** — The default table format for this catalog: Iceberg, Mixed-Iceberg, Mixed-Hive, or Paimon.
    - **Storage** — File system configuration (HDFS, S3, OSS, etc.) and credentials.

    After saving, the catalog appears in the catalog list and is available to connected compute engines.

    For detailed catalog configuration options, see [Managing Catalogs](/admin-guides/managing-catalogs).
  </Step>

  <Step title="Create your first table">
    With a catalog created, open the **Terminal** in the Dashboard. Navigate to **Terminal** in the left sidebar and run:

    ```sql
    CREATE DATABASE IF NOT EXISTS my_db;

    CREATE TABLE my_catalog.my_db.my_table (
      id BIGINT,
      name STRING,
      ts TIMESTAMP
    ) USING iceberg
    PARTITIONED BY (days(ts));
    ```

    After creation, the table appears under **Tables** in the Dashboard, where you can view its schema, snapshots, files, and self-optimizing status.
  </Step>
</Steps>
